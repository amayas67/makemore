Makemore Part 3 - First Optimizations

After building a basic MLP in Makemore Part 2, Karpathy begins to improve the training process. The architecture does not change yet. Instead, he focuses on making optimization easier.

1. Initializing W2 with a Small Scale

W2 = torch.randn((n_hidden, vocab_size)) * 0.01

Instead of using the raw values produced by torch.randn(), every weight is multiplied by 0.01.

Why?

The output logits are computed as:

logits = h @ W2 + b2

If W2 contains large values, the logits can immediately become large.

The softmax applies an exponential:

softmax(x) = exp(x) / Σ exp(x)

The exponential amplifies differences. Large logits produce extremely confident probabilities even before learning.

Example

Logits:

[5.2, -2.8, 1.4]

↓

exp()

[181, 0.06, 4.05]

↓

Softmax

[0.978, 0.0003, 0.021]

The network is already almost certain about one class despite having learned nothing. This is undesirable.

By multiplying W2 by 0.01, the logits remain close to zero. 

Logits:

[0.03, -0.02, 0.01]

↓

exp()

[1.030, 0.980, 1.010]

↓

Softmax

[0.34, 0.32, 0.34]

The initial probabilities become much more uniform. The network starts without any strong preference, which is a much healthier initialization.

Important: The multiplication by 0.01 does not change the fact that the weights follow a normal distribution. Instead of N(0,1) the weights now follow approximately N(0,0.01²) The distribution is simply much narrower around zero.

2. Tanh Saturation and the Vanishing Gradient

The hidden layer uses the tanh activation: 

h = torch.tanh(...)

Its derivative is:

tanh'(x) = 1 − tanh²(x)

If we denote

t = tanh(x)

then

dt/dx = 1 − t²

What happens near ±1?

Suppose 

tanh(x) = 0.999

Then

1 − 0.999² ≈ 0.002

The derivative is almost zero.

Backpropagation

During backpropagation the chain rule multiplies gradients. 

Incoming gradient

1

×

Derivative of tanh

0.002

=

Outgoing gradient

0.002

Only a tiny gradient reaches the previous operations.

The weights before tanh therefore receive almost no update:

W = W − learning_rate × gradient

If the gradient is almost zero, the weight update is almost zero as well.

This phenomenon is called Vanishing Gradient. As the gradient travels backward through the network, it becomes smaller and smaller, making earlier layers learn extremely slowly.

Why does initialization matter?

If the weights are initialized with very large values, the input of tanh becomes large as well. 

x = 7

↓

tanh(x) ≈ 1

↓

Derivative ≈ 0

The neuron starts immediately in the saturated region. Learning becomes difficult from the very first iteration.

A better initialization keeps the activations close to zero, where tanh behaves almost linearly. 

x = 0

↓

tanh(0) = 0

↓

Derivative = 1

This allows gradients to flow much more easily through the network.

Key Takeaways

  • Initializing W2 with *0.01 keeps the initial logits small.
  • Small logits produce nearly uniform probabilities.
  • The derivative of tanh is 1 - tanh²(x).
  • When tanh saturates near ±1, its derivative approaches zero.
  • Very small derivatives produce the Vanishing Gradient.
  • Good initialization helps keep neurons away from saturation during the first training iterations.
